ATERAMEDIA PASSWORDS MANAGER — Gestor de Credenciales
========================================================

Este paquete es una aplicación web autónoma (no necesita instalar nada,
ni conexión a internet, ni servidor). Todas las librerías y tipografías
están incluidas dentro de la carpeta "vendor" y "icons".

────────────────────────────────────
NOVEDADES DE ESTA VERSIÓN (IMPORTANTE)
────────────────────────────────────
Se corrigió un bug real de programación (no era un problema de caché
ni de subida de archivos, aunque lo parecía). El botón "Pegar contenido
manualmente" de Ajustes intentaba conectarse apenas cargaba la página,
pero ese botón en realidad se crea recién al entrar a la sección de
Ajustes (se genera dinámicamente). Eso hacía que el script fallara
apenas arrancaba, dejando sin conectar TODOS los botones que se
registran después en el código — por eso la clave maestra funcionaba
pero crear credenciales, grupos, etc. no respondía.

Se verificó con una prueba automatizada real (no solo lectura de
código): se simuló crear un vault, guardar una credencial, crear un
grupo, navegar por todas las secciones y usar los botones de
sincronización — todo sin errores.

Si ya habías subido una versión anterior con este problema, sube esta
para corregirlo definitivamente.

────────────────────────────────────
1) USO MÁS SIMPLE (cualquier PC/Mac)
────────────────────────────────────
Haz doble clic en "index.html". Se abrirá en tu navegador
(Chrome, Safari, Edge o Firefox) y funcionará 100% offline,
sin necesidad de instalar nada. Los datos quedan guardados en
la base de datos local del navegador (IndexedDB) para la próxima vez.

────────────────────────────────────
2) INSTALAR COMO APP EN EL CELULAR
────────────────────────────────────
Los navegadores de Android/iOS no permiten instalar apps directamente
desde un archivo local: la carpeta debe servirse desde una URL
(http/https). Dos caminos gratuitos:

  OPCIÓN A — Hosting estático gratuito (ej. GitHub Pages)
  ---------------------------------------------------------
  Sube TODA la carpeta "AteraMediaPasswordsManager" (usa "Add file →
  Upload files" arrastrando los archivos tal cual, no los copies y
  pegues a mano) a GitHub Pages, Netlify Drop, Vercel o Cloudflare
  Pages. Luego, desde el celular:
    - Android (Chrome): menú (⋮) → "Instalar aplicación".
    - iPhone (Safari): compartir (□↑) → "Añadir a pantalla de inicio".

  OPCIÓN B — Servidor local en tu red (sin subir nada a internet)
  -------------------------------------------------------------------
      cd AteraMediaPasswordsManager
      python -m http.server 8080
  Y entra desde el celular a http://IP_DE_TU_PC:8080

────────────────────────────────────
3) DÓNDE VIVEN LOS DATOS
────────────────────────────────────
• Cada dispositivo guarda sus datos por separado en su propia base de
  datos local (IndexedDB) — no hay servidor central.
• Si borras los datos de navegación del sitio, o desinstalas la app,
  los datos de ese dispositivo se pierden (todo vive cifrado y local).
• Usa "Informes" para exportar un PDF de respaldo, y "Sincronizar
  entre dispositivos" (punto 5) para mantener PC y celular al día.

────────────────────────────────────
4) SEGURIDAD
────────────────────────────────────
• Los datos se cifran con AES-256 (clave derivada por PBKDF2, 120.000
  iteraciones) antes de guardarse en IndexedDB. La clave maestra nunca
  se guarda en texto plano ni se envía a ningún servidor.
• Si olvidas tu clave maestra, los datos no se pueden recuperar.

────────────────────────────────────
5) SINCRONIZAR ENTRE DISPOSITIVOS (PC ↔ CELULAR)
────────────────────────────────────
En "Ajustes → Sincronizar entre dispositivos":
  • EXPORTAR: descarga un backup cifrado (.skvault).
  • IMPORTAR Y FUSIONAR: agrega lo nuevo del otro dispositivo sin
    borrar nada; si una misma credencial cambió en ambos lados, gana
    la más reciente.
  • Si el archivo aparece "ilegible" en iPhone: usa "pegar el
    contenido manualmente" — abre el .skvault en Archivos o Notas,
    copia todo el texto, y pégalo directo en la app.
  • Para restaurar en un dispositivo nuevo: en la pantalla inicial,
    "¿Ya tienes un backup? Importar vault existente".

────────────────────────────────────
6) SI ALGO NO RESPONDE: cómo revisar la consola
────────────────────────────────────
Si algún botón deja de responder:
  1. Abre la app en una computadora, presiona F12 → pestaña "Console".
  2. Copia el mensaje de error completo tal cual aparece (incluye el
     nombre del archivo, línea y columna).
  3. Ese mensaje indica exactamente qué elemento o función falló —
     es la forma más rápida de diagnosticar cualquier problema futuro.

────────────────────────────────────
7) DESPLEGAR EN RAILWAY (servidor zero-knowledge)
────────────────────────────────────
Este paquete incluye ahora un pequeño servidor (carpeta "server/") que permite
sincronizar el vault cifrado entre dispositivos vía API. El servidor NUNCA recibe
tu clave maestra: solo guarda el vault ya cifrado (AES-256 + PBKDF2 120.000).

Requisitos: Node.js 22+ (en Railway se fija con NODE_VERSION=22).
Pasos:
  1. Sube TODO el paquete a un repo de GitHub (la raíz del repo debe contener
     "railway.json" y la carpeta "server/").
  2. En Railway: New Project → Deploy from GitHub repo → elige el repo.
  3. Railway lee "railway.json" automáticamente:
       • rootDirectory: server
       • build: nixpacks (npm install)
       • start: node server.js
       • Variables: PORT=3000, NODE_VERSION=22, DB_PATH=/data/supreme_key.db
       • Volumen montado en /data (SQLite persistente; el disco de la app es efímero).
  4. Deploy. Railway te da una URL https pública. ¡Listo!

En el navegador, entra a la URL, crea tu clave maestra (como siempre) y luego en
"Ajustes → Sincronizar entre dispositivos (nube)" crea una cuenta con tu correo.
Tu vault queda guardado cifrado en el servidor y puedes recuperarlo desde cualquier
otro dispositivo con la misma clave maestra. El respaldo manual por .skvault sigue
disponible como método alternativo sin servidor.

Notas de seguridad del servidor:
  • La cookie de sesión es httpOnly + Secure + SameSite=Lax (solo en HTTPS).
  • El servidor solo almacena salt, verifier y el vault cifrado; jamás la clave.
  • Si olvidas la clave maestra, el servidor tampoco puede descifrar tus datos.
